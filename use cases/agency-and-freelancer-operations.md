# Agency And Freelancer Operations

## Retainer Automation
- Run a standard technical health check for every client on the first crawl of the month.
- Publish a rolling backlog of new 4xx pages, redirect chains, broken canonicals, and noindex regressions.
- Track issue deltas week over week so retainers focus on changes, not static inventory.
- Produce fixed-format issue packs for account managers without opening the GUI.

## Delivery Standardization
- Define one crawl intake contract for every client regardless of site size.
- Reuse the same Python report pipeline across local, cloud, and collaborator environments.
- Validate crawl completeness before analysts start writing recommendations.
- Turn raw crawl output into repeatable service lines instead of analyst-specific spreadsheets.

## Prioritization Workflows
- Rank issues by affected URLs, section, template, or page type.
- Separate sitewide defects from isolated URL exceptions before delivery.
- Route content issues to content teams and rendering/indexability issues to engineering.
- Build client-specific threshold rules for alerting vs reporting.

## Margin And Throughput
- Replace manual exports with reusable scripts that run on every new crawl.
- Store crawl snapshots for re-analysis without rerunning Screaming Frog.
- Create internal QA checks so junior analysts can ship consistent reports faster.
- Package common audits as repeatable internal tools or MCP actions.
