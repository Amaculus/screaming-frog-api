# Migration War Room

## Pre-Migration Baselines
- Snapshot titles, descriptions, canonicals, headings, status codes, and internal link metrics before cutover.
- Build protected URL inventories for critical sections and revenue-driving pages.
- Identify redirect chains and canonical issues before adding migration complexity.
- Freeze a reproducible crawl artifact for later comparisons.

## Cutover Monitoring
- Run fast post-launch crawls on priority sections.
- Compare current status, titles, canonicals, indexability, and link support against baseline.
- Detect routing failures, redirect mistakes, and template regressions immediately.
- Surface new orphaned or weakly linked pages after navigation changes.

## Stabilization
- Re-run focused diffs daily during migration cleanup.
- Track whether fixes actually reduce defect counts rather than reshuffle them.
- Separate legacy debt from migration-introduced problems.
- Build remediation queues by issue family and owning team.

## Executive And Stakeholder Reporting
- Publish concise what-changed reports during the migration window.
- Quantify URL-level and section-level impact from technical regressions.
- Keep evidence attached to every major finding.
- Maintain one source of truth for migration QA instead of ad hoc exports.
