# SaaS And Product-Led Growth

## App Surface QA
- Audit docs, marketing, blog, pricing, and app-adjacent pages in one crawl model.
- Detect internal linking gaps between conversion pages and educational content.
- Monitor technical regressions on release-note, integration, and template-heavy sections.
- Track indexability and metadata quality across rapidly changing product content.

## Growth Page Governance
- Validate signup, pricing, comparison, and feature pages after launches.
- Detect redirect, canonical, and title regressions on bottom-funnel URLs.
- Compare crawl changes for launch weeks versus stable periods.
- Build protected page sets for conversion-critical URLs.

## Documentation And Help Centers
- Audit versioned docs, integration directories, and API reference sections.
- Detect orphaned or weakly linked docs after IA changes.
- Track duplicate or semantically overlapping support content.
- Build internal-linking opportunity lists from docs to product pages.

## Release Coordination
- Attach crawl-diff evidence to product or web release retrospectives.
- Separate content regressions from framework or rendering regressions.
- Run lightweight post-release crawls on changed sections only.
- Feed recurring findings into web platform backlogs.
