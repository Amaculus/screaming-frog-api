Planned build-out:
- automated SEO workflows
- CI/CD and release guardrails
- monitoring and alerting
- client delivery and reporting
- content and information architecture
- migrations and redesigns
- data platform / warehousing
- ecommerce and publishing vertical recipes
- localization / hreflang operations
- productized services and internal tools

Principles:
- prefer `.dbseospider` as the source of truth when possible
- treat Screaming Frog GUI exports as optional, not required
- keep workflows scriptable, testable, and schedulable
- optimize for repeatability, not one-off analysis
