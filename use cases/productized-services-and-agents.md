# Productized Services And Agents

This file focuses on packaging repeatable workflows into internal tools, services, or agent systems.

## Productized service ideas
- monthly technical health monitoring
- migration QA retainers
- redirect debt cleanup service
- content decay monitoring
- hreflang operations retainer
- release-guardrail setup for product teams

## Internal tool ideas
- crawl triage dashboard
- template governance console
- launch QA command runner
- client-facing health portal
- issue routing system by owner or market

## Agent use cases
- agent that investigates a single broken destination and returns source URLs, anchors, and likely fix paths
- agent that turns a crawl diff into a weekly remediation memo
- agent that opens Jira issues for new blockers only
- agent that answers ad-hoc technical SEO questions from raw SQL
- agent that prepares migration QA punch lists from before-and-after crawls

## Why the library matters here
- typed workflows cover the common tasks
- raw SQL covers the edge cases
- `.dbseospider` keeps the workflow independent from manual GUI export steps
