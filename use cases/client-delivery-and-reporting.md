# Client Delivery And Reporting

This file focuses on turning crawl data into deliverables for clients or internal stakeholders.

## Reporting patterns
- Monthly health check report.
- Release regression summary.
- Migration QA report.
- Redirect cleanup backlog.
- Metadata optimization backlog.
- International SEO issue summary.

## Useful sections in a recurring report
1. Executive summary.
2. What changed since the previous crawl.
3. New blockers.
4. Issue counts by category.
5. Priority remediation list.
6. Appendix with URLs and examples.

## Good report inputs
- crawl diff change sets
- status code breakdowns
- indexability summaries
- redirect and canonical chain reports
- top broken destinations by number of linking pages
- top templates with metadata issues
- accessibility and structured-data summaries

## Why the library helps
- The deliverable can be regenerated at any time.
- The same methodology applies across clients.
- Analysts spend less time exporting and cleaning CSVs.
- Raw SQL remains available for custom client questions.
