# Experimentation And Forecasting

## Controlled Change Analysis
- Compare crawls before and after title, description, or canonical experiments.
- Measure whether technical changes altered internal link support for target pages.
- Build template-level regression reports for A/B or phased rollouts.
- Track whether experimental sections accumulate new crawl-depth or indexability issues.

## Forecast Inputs
- Estimate remediation backlog size by template, section, or issue family.
- Quantify how many URLs would be affected by a proposed rule change before rollout.
- Build page-type inventories for scenario planning.
- Track issue creation and resolution rates over time.

## Program Tracking
- Use crawl diffs as operational evidence that a rollout changed the intended fields only.
- Detect spillover effects into unrelated templates.
- Separate deliberate edits from accidental regressions.
- Produce analyst-ready summaries for what changed, where, and how much.

## Strategy Support
- Prioritize cleanup programs by issue concentration and page importance.
- Identify sections with chronic regressions rather than isolated incidents.
- Build technical baselines before migrations, redesigns, or large content operations.
- Turn crawl history into a forecasting input for engineering planning.
