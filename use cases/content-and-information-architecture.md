# Content And Information Architecture

This file focuses on content quality, internal structure, and inventory planning.

## Core workflows
- find thin content in important directories
- find near-duplicate and semantically similar pages
- find pages with weak internal link support
- find pages with stale or template-only metadata
- find pages where titles, H1s, and canonicals disagree
- map crawl depth against business importance

## Backlog generation patterns
- editorial rewrite queue
- metadata rewrite queue
- internal-linking improvement queue
- consolidation queue for overlapping pages
- deindex or merge review queue for thin pages

## Information architecture workflows
- identify sections with poor discovery depth
- compare hub pages before and after navigation changes
- track unique inlink loss on strategic category pages
- model path-based linking between sections

## Why code helps
- the same prioritization logic can run every crawl
- teams can join crawl data with business priority inputs
- content planning becomes a reproducible system, not a one-off spreadsheet
